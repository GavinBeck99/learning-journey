## üí° Key Learnings

### Git & GitHub
- **Git** is version control - tracks code changes over time
- **GitHub** is where code lives online (like Google Drive for code)
- **SSH keys** = secure connection without passwords every time
- `git config` sets up my identity for commits
- My commits will show as "Gavin Beck" in history

### AI-Assisted Development ("Vibe Coding")
- AI can generate entire components from natural language descriptions
- "Vibe coding" = describing what you want, AI writes the code
- Professional code includes detailed comments and documentation
- TypeScript adds type safety to JavaScript (catches errors before they happen)
- Accessibility (ARIA attributes) should be built into all components
- Effective prompting is a skill - simpler prompts often work better

### TypeScript
- TypeScript is JavaScript with type checking
- Prevents bugs by catching mistakes while coding (not when app runs)
- Types like `'primary' | 'secondary' | 'danger'` limit allowed values
- Interface defines what props a component accepts
- Makes code self-documenting and easier to maintain

### React Components
- Components are reusable pieces of UI
- Props pass data into components
- Children prop contains what's inside the component tags
- Default values make components easier to use
- Conditional rendering shows/hides elements based on state

### Development Environment
- **Terminal/PowerShell** = text-based computer control
- **Node.js** = runs JavaScript outside the browser
- **npm** = package manager (installs tools and libraries)
- **Cursor** = VS Code with AI capabilities built-in
- After installing software, sometimes need to restart terminal/computer

## üéì Professional Patterns Observed

### In the Generated Button Component
- **Type Safety:** TypeScript interfaces prevent mistakes
- **Default Props:** `variant = 'primary'` makes component easier to use
- **Accessibility:** `aria-*` attributes for screen readers
- **Separation of Concerns:** Helper functions for class names
- **Documentation:** Comments explain WHY, not just WHAT
- **Flexibility:** `{...rest}` allows passing additional props
- **Loading States:** UX consideration for async operations
- **Disabled States:** Prevents user errors during operations

### What Makes Code "Professional"
- Proper TypeScript types
- Accessibility built-in
- Detailed comments
- Consistent naming conventions
- Error prevention (disabled during loading)
- Reusability through props
- Documentation for future developers
